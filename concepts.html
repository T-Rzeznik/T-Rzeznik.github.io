Arrays in javascript are mutable which means that they are able to be changed or mutated after initialization. You are able to mutate arrays through functions such as splice() which removes a section of the array and replaces it with new elements, slice() which returns a subset of the array. Other functions that allow the user to work with arrays are indexOf() which will return the index of a specified element, and filter which creates a new array based on a boolean value, an example of this would be filtering all the even or odd numbers from an array.

As stated in the answer above arrays are mutable, but arrays can also work as a stack or a queue. Since queues and stacks function differently (LIFO and FIFO) there are different functions used to treat arrays as such. For a queue, an array will work with shift(), and push(). Shift is a function that removes the first element from the array. Push is a function that adds one element to the end of the array. With these 2 functions you can easily implement a queue in javascript. To implement a stack in javascript you can use functions push() and pop(). Pop will remove the last element in the array, while push again will add an element to the end of an array. With these functions you can easily implement a stack in javascript.

Asynchronous: Async functions and programming allows js functions to execute without waiting for the entire page to load and allows for functions to be executed at the same time. With async functions webpages are more efficient.

Synchronous: Synchronous programming is when code executes one after the other like a queue. JavaScript is synchronous by default and needs the use of async and await functions to become asynchronous. Synchronous pages tend to take more time to load as the page must wait to execute each function in line.

Await: Await is a part of async programming. When you use await you wait for an async function to return a promise before executing. While waiting for the promise the function is “paused” and does not execute until a promise is resolved or rejected.

Callback Functions: Callback functions are functions that are used as parameters in other functions. They are used in async programming to handle the result of an async function.

Blocking calls: Blocking calls are in synchronous programming to stop executing of code until the previous function is done running.

Non-Blocking calls: Non Blocking calls are in async programming to continue executing functions while other functions are being executed.

-POST is used to send data to the server and create a new instance with the data.
-PUT is used to update existing data in the server
-DELETE is used to delete data in the server

Request and response messages: A HTTP request is when the client is requesting data from the server. A request will include what type of request it is such as a GET request, along with the url being fetched from, and a header of information to go with the get request. A Response message is when the server responds to a request from the client. Response messages often come with a note saying if the request was fulfilled or failed.

HTTP Message header fields: These header fields are used in both request and response messages and are used to provide information or context for what is being requested/responded.

HTTP Response status codes: These are codes that come along with responses from the server. They provide information on the response in regards to failure, success, or any other issues that may have happened. 404 Not Found is an example of this.
